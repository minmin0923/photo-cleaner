# PhotoCleaner — 맥 없이, 0원으로 아이폰에 설치하기

같은 배경·포즈의 중복 사진을 찾아 가장 잘 나온 사진을 추천하고, **1장/2장 남기기**를 선택해 정리하는 iOS 앱. 맥 없이 GitHub Actions(무료)로 빌드하고, Windows PC의 Sideloadly(무료)로 설치합니다.

## 안전 설계 (오삭제 방지 4중 장치)

| 단계 | 장치 |
|---|---|
| 1 | 앱은 스스로 아무것도 삭제하지 않음 — 사용자가 그룹별로 확정한 사진만 대상 |
| 2 | **안전 모드 기본 ON** — 삭제 대신 "🗑 삭제후보 (PhotoCleaner)" 앨범에 모으기만. 실제 삭제는 사진 앱에서 직접 |
| 3 | 최종 확인 화면 — 처리 대상 전체를 다시 보여주고 체크 확인 후에만 실행 |
| 4 | 직접 삭제 모드에서도 iOS 시스템 확인창 + '최근 삭제됨' 30일 복구 |
| + | iCloud 로드 실패 사진은 분석 자체에서 제외 → 어떤 경우에도 건드리지 않음 |

## 1단계 — GitHub 레포 만들기 (5분)

1. github.com → New repository → 이름 예: `photo-cleaner` → **Public** 선택 (공개 레포는 Actions 무제한 무료)
2. 이 폴더의 파일을 **구조 그대로** 업로드:

```
photo-cleaner/
├─ project.yml
├─ .github/workflows/build.yml
└─ Sources/
   ├─ PhotoCleanerApp.swift
   ├─ ScanViewModel.swift
   ├─ QualityScorer.swift
   ├─ ContentView.swift
   ├─ GroupReviewView.swift
   └─ FinalConfirmView.swift
```

3. 업로드(커밋)하면 Actions가 자동 실행됨. 레포의 **Actions 탭**에서 진행 확인 (약 5~10분)
4. 초록 체크 완료 후 해당 실행 클릭 → 하단 **Artifacts** → `PhotoCleaner-unsigned-ipa` 다운로드 → 압축 풀면 `PhotoCleaner-unsigned.ipa`

> 빌드가 빨간 X로 실패하면: 실패한 스텝의 로그 전체를 복사해서 Claude에게 붙여넣으면 수정해 드립니다. (이 첫 실행이 곧 컴파일 검증 단계입니다)

## 2단계 — Windows에서 아이폰에 설치 (10분)

1. Apple 공식 사이트에서 **iTunes**와 **iCloud** 설치 (⚠️ Microsoft Store 버전 말고 apple.com 웹 다운로드 버전)
2. sideloadly.io 에서 Sideloadly 설치
3. 아이폰을 USB로 연결 → Sideloadly 실행 → IPA 파일을 창에 드래그
4. Apple ID 입력 (2단계 인증 코드 입력 필요. 걱정되면 사이드로드 전용 보조 Apple ID를 만들어 써도 됨)
5. Start → 완료 후 아이폰에서 *설정 > 일반 > VPN 및 기기 관리* → 본인 Apple ID **신뢰**
6. 앱 실행

### 무료 Apple ID 제약 (미리 인지)

| 제약 | 내용 | 대응 |
|---|---|---|
| 7일 서명 | 7일 지나면 앱이 안 열림 | Sideloadly 자동 재서명 기능 사용, 또는 주 1회 PC 연결해 재설치 (데이터는 유지됨) |
| 기기당 앱 3개 | 사이드로드 앱 동시 3개까지 | 이 앱 하나면 문제 없음 |

## 3단계 — 첫 사용 검증 절차 (필수 권장)

실사진에 바로 쓰지 말고 아래 순서로 검증하세요:

1. **테스트 데이터 만들기**: 같은 장면을 연속 4~5장 촬영 (일부러 눈 감은 컷, 흔들린 컷 포함)
2. 앱에서 `최근 30일` + `안전 모드 ON`으로 스캔
3. 확인할 것: ① 방금 찍은 사진들이 한 그룹으로 묶이는가 ② 눈 감은 컷 점수가 낮은가 ③ 관련 없는 사진이 그룹에 섞이지 않는가
4. 그룹에서 1장/2장 선택 → 최종 확인 → 실행 → 사진 앱에서 "🗑 삭제후보" 앨범 확인 (**원본이 그대로 있는지 확인**)
5. 여기까지 이상 없으면 기간을 넓혀 사용. 직접 삭제 모드는 안전 모드로 2~3회 써본 뒤 전환 권장

### 판정이 이상할 때 튜닝

| 증상 | 조정 |
|---|---|
| 다른 장면이 한 그룹으로 묶임 | 유사도 민감도 0.30~0.35로 낮춤 |
| 명백한 중복인데 안 잡힘 | 민감도 0.50~0.60으로 올림 |
| 시간 간격 둔 중복을 못 잡음 | 촬영 간격 300~600초로 확대 |

## 작동 원리 요약

- **그룹핑**: 촬영 간격 90초 이내 + Vision FeaturePrint 시각 유사도 (기본 임계값 0.40, 보수적)
- **채점**: 인물 사진 = 얼굴 촬영 품질 55% + 눈 뜸 20% + 선명도 20% + 노출 5% / 풍경 = 선명도 70% + 노출 30%
- 100% 온디바이스 분석, 외부 전송 없음

## 알려진 한계 (v2 기준)

- 피사체 끼어듦(포토바밍)은 간접 감지 (유사도·점수 하락으로 반영). 직접 감지는 v3 과제
- 선글라스·측면 얼굴은 눈 감음 판정 제외 (오탐 방지 목적)
- FeaturePrint 거리 스케일이 iOS 버전별로 달라 첫 스캔 후 민감도 보정이 필요할 수 있음
